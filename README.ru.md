# GameAP GoldSource Addons

Плагин GameAP для управления Metamod и AMX Mod X на GoldSource-серверах
(Half-Life 1, Counter-Strike 1.6 и др.). Добавляет на страницу сервера вкладку
«Плагины»: статус платформ, список плагинов из `plugins.ini` с группировкой по
секциям, включение/выключение, флаг `debug` и комментарии у плагинов, удаление,
установка из файла и редактирование конфигов.

- **Бэкенд** — WASM-модуль на Rust ([gameap-plugin-sdk](https://github.com/gameap/gameap-proto)):
  собирает состояние сервера и атомарно правит `plugins.ini` / читает
  `liblist.gam` через host-библиотеку `nodefs`.
- **Фронтенд** — Vue 3 + naive-ui, встроен в WASM-модуль. Использует
  существующие эндпоинты панели: RCON (`meta version`, `amxx version`,
  `meta list`, `amxx plugins` — версии и runtime-статусы), файловый менеджер
  (загрузка файлов плагинов, конфиги), перезапуск сервера.

## Возможности

- Карточки Metamod / AMX Mod X: установлен · не активен · не установлен,
  версия из консоли сервера, счётчики плагинов.
- Таблица плагинов (AMXX и Metamod): статус (Работает / Ждёт перезапуска /
  Ошибка `bad load` / Файл отсутствует / Выключен), версия и автор из консоли,
  поиск, фильтр, массовые действия.
- Группировка по секциям `plugins.ini` (`; Basic`, `; Menus`, …): каждая
  именованная секция становится группой; плагины вне заголовков (одиночные и
  безымянные блоки) собираются в общую группу «Прочее». Если секции распознать
  не удаётся, список показывается плоским.
- Комментарии к плагинам (inline `; ...` после записи) отображаются и
  редактируются; флаг загрузки `debug` у плагинов AMX Mod X отображается и
  переключается.
- Включение/выключение — комментирование строки в `plugins.ini`
  с побайтовым сохранением остального файла (CRLF, BOM, CP1251-комментарии).
- Пауза/снятие с паузы AMXX-плагина из списка — консольные команды
  `amxx pause <file>` / `amxx unpause <file>` через RCON, применяется
  мгновенно, без перезапуска; кнопка доступна, когда сервер онлайн и RCON
  работает.
- Установка из файла: `.amxx` для AMX Mod X, `.so`/`.dll` для Metamod
  (файл загружается через файловый менеджер, затем регистрируется в ini).
  `.sma` отклоняется с подсказкой скомпилировать. Повторная загрузка уже
  установленного плагина перезаписывает файл новой версией (`force`),
  строка в `plugins.ini` остаётся на своём месте.
- Редактирование конфига плагина (`configs/<имя>.cfg|.ini`) в модальном окне.
- Баннер «Требуется перезапуск» с кнопкой перезапуска сервера.

Установка/обновление/удаление самих Metamod и AMX Mod X в плагин не входит.

## Видимость вкладки

Вкладка показывается только:

- на серверах с движком **GoldSource** (`game.engine == "GoldSource"`) —
  через `checkGame` слота `server-tabs` (нужен gameap-api с поддержкой
  `checkGame`, plugin-sdk ≥ 0.3.3);
- пользователям с ability `plugin:ezvdsxmlu6fbk:manage` — админы получают её
  автоматически. Не выдавайте её обычным пользователям: HTTP-маршруты плагина
  объявлены `admin_only`, не-админ увидит вкладку, но получит 403.

## Сборка

Требуется Rust (toolchain и target зафиксированы в `rust-toolchain.toml`:
1.94.0 + `wasm32-wasip1`), Node.js и локальная копия
[gameap-proto](https://github.com/gameap/gameap-proto) рядом с этим
репозиторием (path-зависимость `../gameap-proto/rust/gameap-plugin-sdk`),
а также gameap-api рядом (`../gameap-api` — npm-зависимость
`@gameap/plugin-sdk`).

```sh
make build     # frontend (npm ci && vite build) + wasm; артефакт goldsrc-addons.wasm
make test      # cargo test (ini/liblist/handlers) + vitest (парсеры RCON)
make lint      # clippy для native и wasm32-wasip1
```

Порядок важен: фронтенд собирается до wasm — `build.rs` встраивает
`frontend/dist/plugin.js` в модуль (без dist соберётся модуль без UI).

## Установка

Скопируйте `goldsrc-addons.wasm` в каталог плагинов панели (или загрузите
через Администрирование → Плагины) и перезапустите панель.

API плагина: `/api/plugins/ezvdsxmlu6fbk/...`

| Метод | Путь | Назначение |
|---|---|---|
| GET | `/servers/{id}/state` | Состояние Metamod/AMXX и списки плагинов |
| POST | `/servers/{id}/{platform}/plugins/toggle` | `{file, enabled}` — вкл/выкл записи |
| POST | `/servers/{id}/{platform}/plugins/attributes` | `{file, debug, comment}` — задать флаг `debug` и inline-комментарий (AMXX; для Metamod применяется только комментарий/описание, `debug` игнорируется) |
| POST | `/servers/{id}/{platform}/plugins` | `{file, enable, path?, force?}` — регистрация загруженного файла; `force: true` перезаписывает существующую запись (200 вместо 409) |
| DELETE | `/servers/{id}/{platform}/plugins` | `{file}` или `?file=` — удалить запись и файл |

`{platform}` — `amxx` либо `metamod`; все маршруты требуют администратора.

## Заметки

- Id плагина — `ezvdsxmlu6fbk` (без дефиса): панель нормализует id через
  `CompactPluginID`, и дефисные id переписываются в hash, ломая пути API.
- Правки `plugins.ini` — read-modify-write без блокировок (last-write-wins);
  параллельное редактирование файла другим способом может быть перезаписано.
  Включение/выключение сохраняет остальной файл побайтно; изменение флага
  `debug` или комментария записи переписывает только эту одну строку (её
  пробелы нормализуются).
- Версии и статусы из консоли доступны только когда сервер запущен и задан
  RCON-пароль; без них статусы показываются по `plugins.ini`.
- Диагностика консоли: неверный RCON-пароль (ответ «Bad Password» от
  GoldSource / HTTP 422 от панели) и неожиданно пустой ответ консоли
  показываются отдельной подсказкой вместо молчаливых пустых статусов.
